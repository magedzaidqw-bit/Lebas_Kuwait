# لباس الكويت (Lebas Kuwait)

نظام إدارة عمليات جمع الملابس المستعملة — شركة زيرو ويست للتجارة العامة والمقاولات.
مبني بنفس نمط مشروعي كيان والتراث: تطبيق PWA بملف واحد + Supabase.

## البوابات
- **مدير**: لوحة تحكم شاملة، إدارة المناديب.
- **مندوب**: تسجيل عمليات الاستلام الميدانية (وزن، نوع الطلب، بيانات العميل).
- **مشرف**: تسجيل الوزن الفعلي بالمخزون ومقارنته تلقائيًا بما سجّله المندوب (كشف التلاعب).
- **محاسب**: عهدة كل مندوب، تسجيل الدفعات.

## قاعدة البيانات
انظر `schema.sql` — يحتوي على الجداول الخمسة (`agents`, `collections`, `warehouse_entries`, `payments`, `app_users`) وواجهتي عرض جاهزتين (`reconciliation`, `agent_balances`).

**خطوات التفعيل:**
1. أنشئ مشروع Supabase جديد.
2. نفّذ محتوى `schema.sql` كاملًا في SQL Editor.
3. انسخ Project URL و anon public key من Settings → API، وأرسلهما لإدخالهما في `config.js`.

⚠️ سياسات RLS الحالية في `schema.sql` مفتوحة (`using (true)`) للتطوير الأولي فقط — يجب تقييدها حسب الدور قبل الاستخدام الفعلي مع الفريق (نفس ما تم مع كيان والتراث).
